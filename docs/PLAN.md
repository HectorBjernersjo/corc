# corc rewrite plan

Outcome of the design interview 2026-07-08, covering every item in
`features_i_want.md`. Companion documents: `docs/CONTEXT.md` (vocabulary,
states, lifecycle) and `docs/adr/0001-hidden-session-swap-pane.md`
(pane-topology decision). This document records **all decisions** and the
**five implementation steps**.

---

## 1. Vision

corc is the single place Claude Code is used from. It owns every Claude
process it shows: it spawns them, parks them in a hidden tmux session, swaps
them into view next to a narrow sidebar, tracks their lifecycle, and jumps to
the user's real working sessions. Claude Code is never started manually;
corc never guesses.

---

## 2. Decisions

### D1 — Ownership model (features 6, 7, 10)

Only conversations spawned by corc exist. No `/proc` scanning, no adoption of
foreign `claude` processes, no discovery of pre-existing `~/.claude/projects`
history. `procs.rs` and the cwd-guessing in `status.rs::annotate` are deleted.

### D2 — Hidden session

One global hidden tmux session named `_corc-sessions`, one window per live
conversation, **window name = conversation uuid**. Same tmux server as the
user's sessions (panes cannot move between servers — see ADR-0001). `new.sh`
is patched to filter out `_`-prefixed sessions. The per-project `.tmux.sh`
hook is never run for the hidden session.

### D3 — Exact bookkeeping via `--session-id`

Every spawn uses `claude --session-id <uuid>` (new) or
`claude --resume <uuid>` (resume), with the uuid generated by corc. The
pane ↔ conversation mapping is therefore exact: window name in `_corc-sessions` +
pane id recorded in the state file.

### D4 — Viewing via `swap-pane` (ADR-0001)

corc's window holds two panes: **sidebar** (the TUI, fixed 40 columns, left)
and **content pane** (right). Viewing a conversation is
`swap-pane -s <claude pane> -t <content pane>`; the previously viewed pane is
parked in the hidden window by the same swap. Nothing is ever destroyed, no
`PaneHome`/`unembed` restore logic, the layout is set once and never resized
again. `join-pane` is gone.

### D5 — State file

`~/.local/state/corc/state.json`, written atomically (temp file + rename):

```json
{
  "projects": ["/home/hector/Projects/corc", "..."],
  "conversations": [
    {
      "id": "<uuid>",
      "cwd": "/abs/path",
      "pane_id": "%42",
      "last_viewed": 1751970000,
      "created_at": 1751960000
    }
  ]
}
```

- `projects` — display order of project groups (D9).
- `pane_id` — null when Dead; used for startup reconciliation.
- `last_viewed` — unix seconds; drives the Unseen/Idle distinction.

Dead conversations survive tmux restarts and reboots and stay resumable.
The jsonl files under `~/.claude` are read for metadata but **never modified
or deleted**.

### D6 — Conversation states (feature 9)

| State | Condition | Look | Time column |
|---|---|---|---|
| **Running** | pane alive, turn in flight | yellow ● | elapsed since turn start (`4m`, `1h12m`) |
| **Unseen** | pane alive, turn completed after `last_viewed` | blue ● | duration the completed turn ran |
| **Idle** | pane alive, turn complete, viewed since | gray ● | empty if < 1h old, else coarse age (`5h`) |
| **Dead** | no pane | hollow ○ | coarse age (`5h`, `3d`), never finer than hours |

- The conversation currently in the content pane counts as continuously
  viewed: it goes straight to Idle, never Unseen, and its `last_viewed`
  follows along.
- Seconds are never shown anywhere (feature 5).
- Known limitation (out of scope): a Claude blocked on a permission prompt is
  mid-turn in the jsonl and shows as Running. Fixable later with a Claude Code
  Notification hook.

### D7 — Turn timing

From the jsonl: turn start = timestamp of the last non-sidechain user message;
turn complete = the `end_turn`/`turn_duration` record. Running shows
`now - turn_start`; Unseen shows `turn_complete - turn_start`.

### D8 — Project naming (features 1, 2)

Group headers show the directory **basename only**. A git worktree (detected
by `.git` being a *file* with a `gitdir:` pointer) shows as
`{repo}/{worktree}`, e.g. `corc/fix-ui`. Branches are never shown.

### D9 — Fixed, user-managed project order (feature 4)

New projects are appended when their first conversation is spawned. The user
reorders with **Move mode**: `V` enters, `K`/`J` move the selected project up/
down, `Esc`/`Enter`/`V` exits. Order persists in the state file and never
changes on its own.

Within a project the order is fixed: newest created at the top, never
re-sorted. `created_at` is immutable, so a row never moves once placed —
status flips and fresh activity leave the order untouched, so the user always
keeps their bearings. Which conversations show is still capped to the most
recently active (D13), but membership changes never reshuffle the survivors.

### D10 — Layout (features 3, 8)

No preview pane at all (`draw_preview` and `tail_messages` deleted). On
startup corc splits its own window: left 40 columns = sidebar, right =
content pane holding a plain shell as placeholder. On quit: swap the viewed
pane home, kill the content pane corc created.

### D11 — Mouse

Mouse capture on: click a row = select + view. Scroll wheel moves the
selection.

### D12 — Lifecycle

- Windows in `_corc-sessions` run `claude` as the pane command (no wrapping shell), so
  the window dies when Claude exits. A vanished uuid-window ⇒ conversation
  becomes **Dead** in state (still listed, hollow, resumable). If it dies
  while swapped in, corc recreates the placeholder content pane on the next
  refresh tick.
- `x` on a live conversation kills its Claude and window (`y/n` confirm if
  Running); `x` on a Dead one removes it from the state file and the list.
- Dead conversations older than a week are hidden by default; `a` toggles
  showing everything.

### D13 — Digit jump (feature 11)

`1`–`9` on a selected row targets the **selected row's project**:

1. Ensure the project's real session exists — same convention as `new.sh`
   (basename, `.` → `_`) including running the `.tmux.sh` hook on creation.
2. Window N missing ⇒ create it at index N (`base-index` is 1). For N=1 the
   window is created running nvim.
3. Window 1 exists on an idle shell prompt ⇒ type `nvim` + Enter. Any busy
   foreground process (checked via `#{pane_current_command}`) is never
   disturbed — just focus.
4. `switch-client` there. corc keeps running in its own session.

### D14 — Directory picker (feature 12)

`n` opens a ratatui-native overlay: `~/.config/tmux/directories.txt` expanded
with `git worktree list --porcelain` per repo, deduped — the same source and
expansion as `new.sh`, ported to Rust. Same word-substring matching as the
existing `/` filter. Selecting spawns a fresh Claude in a new hidden window
and swaps it in immediately; `Esc` cancels. The picker shows directories only
(multiple conversations per project is a normal case).

### D15 — Where corc lives

A **visible** session named `_corc`. New subcommand `corc open`: ensure the
session exists with the TUI running, then `switch-client` to it. Dotfiles get
`bind -n C-q run-shell "corc open"` (C-q is free; C-f keeps `new.sh`).

### D16 — Startup reconciliation

On start: for every conversation in state, verify its `pane_id`/uuid-window
still exists (else mark Dead). A Claude pane stranded outside `_corc-sessions` (corc
crashed mid-view) is identified via the state file's `pane_id` and parked back
into a uuid-named hidden window.

### D17 — CLI surface

- `corc` — the TUI.
- `corc open` — see D15.
- `corc list` — kept, rewritten to read the state file + statuses (for
  scripting/inspection).
- `corc watch` — deleted (debug tool for the guessing logic that no longer
  exists).

### D18 — Keybindings after the rewrite

| Key | Action |
|---|---|
| `j`/`k`/arrows, `g`/`G` | move selection |
| `Enter` / mouse click | view (swap in; respawn with `--resume` if Dead) |
| `n` | directory picker → new conversation |
| `x` | kill live / remove dead (confirm when Running) |
| `V`, then `K`/`J` | move mode: reorder projects |
| `1`–`9` | digit jump to real session window N |
| `a` | show hidden (dead > 1 week) |
| `/` | filter |
| `q` | quit (swap home, remove content pane) |

Removed: `o` (goto — meaningless now; digit jump covers "take me there"),
`r` stays as a manual refresh nicety.

## 3. Implementation steps

Each step leaves the program buildable and usable. Steps 1–4 touch only this
repo; step 5 touches dotfiles.

### Step 1 — Core: ownership, state, swap-pane

The load-bearing rewrite; unblocks everything else.

- New `state.rs`: load/save `state.json` (atomic write), types for
  conversations and project order.
- `tmux.rs` rewritten around the new topology:
  - `ensure_hidden_session()`, `spawn_conversation(dir, resume: Option<Uuid>)`
    → new window in `_corc-sessions` named by uuid, running claude directly (D12).
  - `view(conv)` / `park()` via `swap-pane` (D4).
  - startup reconciliation (D16); self-split layout with placeholder (D10).
  - kept as-is: `session_name_for`, `create_session` + `.tmux.sh` hook,
    `is_executable` (reused by digit jump in step 4).
- `discovery.rs` narrowed: given known uuids + cwds, locate and
  incrementally parse only those jsonl files (title, turn state). The
  full-tree scan, `tail_messages`, first/last-prompt fallbacks go away.
- Deleted: `procs.rs`, `status.rs::annotate` guessing, `draw_preview`,
  `watch`, `o`/goto, `embed`/`unembed`/`PaneHome`, `pane_for_pid`, the 45/55
  preview layout.
- Scaffolding: `n` temporarily spawns in corc's own cwd (real picker in
  step 4) so a first conversation can be created.

**Done when:** a fresh conversation can be spawned, viewed, switched between,
survives corc restart (reconciliation), and quitting restores everything.

### Step 2 — Status model and time display

- `discovery.rs`: track `turn_started_at` / `turn_completed_at` per
  conversation (D7).
- `status.rs` rewritten: derive Running/Unseen/Idle/Dead from window
  existence + turn state + `last_viewed` (D6).
- Sidebar rows: the four looks (yellow/blue/gray/hollow) and the per-state
  time column; `last_viewed` maintained for the viewed conversation and
  persisted on swap/quit.
- Fixed within-project ordering by creation time, newest first (D9).

**Done when:** a finished run turns blue until swapped in, the viewed
conversation never turns blue, and no seconds appear anywhere.

### Step 3 — Lifecycle

- Refresh tick notices vanished uuid-windows → mark Dead (D12); recreate the
  content-pane placeholder if the viewed Claude died.
- `x` with `y/n` confirm for Running; state-file removal for Dead.
- Default hiding of week-old Dead conversations behind `a`.
- `Enter` on Dead respawns via `claude --resume <id>` in a new hidden window.

**Done when:** exiting Claude leaves no zombie window, `x` behaves per state,
and the list stays short by itself.

### Step 4 — Interactions

- Directory picker overlay (D14) replacing the step-1 `n` scaffolding.
- Digit jump (D13), reusing `create_session`/hook logic.
- Move mode `V`/`K`/`J` with persisted order (D9).
- Mouse: click to view, wheel to move selection (D11).

**Done when:** every key in the D18 table works as specified.

### Step 5 — Integration (dotfiles + CLI)

- `corc open` subcommand (D15).
- Dotfiles: `bind -n C-q run-shell "corc open"` in `tmux.conf` (absolute
  path to the binary if `run-shell`'s PATH lacks `~/.cargo/bin`).
- `new.sh`: filter `_`-prefixed sessions out of both the fzf list and the
  existing-session check.
- `corc list` rewritten against the state file; `watch` removed (D17).

**Done when:** `Ctrl+q` from anywhere lands in a running corc, and `Ctrl+f`
never shows `_corc-sessions`.

---

## 4. Out of scope (deliberately)

- Permission-prompt detection (Notification hook) — future feature once the
  base is stable.
- Multiple simultaneous content panes / split views.
- Managing or migrating pre-existing `~/.claude/projects` history.
